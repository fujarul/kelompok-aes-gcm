🔑 AES-128 CBC (Cipher Block Chaining) AES-128 → artinya panjang kunci 128 bit (16 karakter ASCII). CBC Mode → setiap blok plaintext akan di-XOR dengan blok sebelumnya sebelum dienkripsi. IV (Initialization Vector) → nilai awal (di sini dibuat tetap, bukan acak). ⚙ Langkah Enkripsi Input: Plaintext → pesan asli (contoh: "HELLO WORLD!!") Key → 16 karakter (contoh: "abcdefghijklmnop") IV → tetap, "1234567890abcdef" Konversi ke Biner/Hex Setiap karakter plaintext dan key diubah ke ASCII → biner (8 bit per karakter). Contoh: "H" = 0x48 = 01001000 "e" = 0x65 = 01100101 Pecah jadi blok 128-bit (16 byte) Jika plaintext > 16 byte, dibagi beberapa blok. Jika kurang → ditambah padding (PKCS7). Blok pertama Plaintext[0] XOR IV Hasil XOR → masuk ke AES-128 Encrypt(Key) Hasilnya = Ciphertext[0] 👉 Rumus: 𝐶 1 = 𝐸 𝐾 𝑒 𝑦 ( 𝑃 1 ⊕ 𝐼 𝑉 ) C 1 ​ =E Key ​ (P 1 ​ ⊕IV) Blok berikutnya Plaintext[i] XOR Ciphertext[i-1] Hasil → dienkripsi → Ciphertext[i] 👉 Rumus umum: 𝐶 𝑖 = 𝐸 𝐾 𝑒 𝑦 ( 𝑃 𝑖 ⊕ 𝐶 𝑖 − 1 ) C i ​ =E Key ​ (P i ​ ⊕C i−1 ​ ) Gabung semua ciphertext → hasil akhir ditampilkan dalam Base64. 🔓 Langkah Dekripsi Input: Ciphertext, Key, IV. Pecah ciphertext ke blok-blok 128 bit. Dekripsi tiap blok dengan key → dapat hasil sementara. XOR dengan IV (untuk blok pertama) atau ciphertext blok sebelumnya (untuk blok selanjutnya). Dapatkan kembali plaintext asli. 👉 Rumus Dekripsi: 𝑃 𝑖 = 𝐷 𝐾 𝑒 𝑦 ( 𝐶 𝑖 ) ⊕ 𝐶 𝑖 − 1 P i ​ =D Key ​ (C i ​ )⊕C i−1 ​ dengan syarat 𝐶 0 = 𝐼 𝑉 C 0 ​ =IV. 📌 Contoh sederhana di papan tulis Misalkan: Plaintext: "HELLO WORLD!!" Key (16 char): "abcdefghijklmnop" IV: "1234567890abcdef" Ubah ke hex: "H" → 0x48, "e" → 0x65, dst. XOR plaintext blok pertama dengan IV. Jalankan AES-128 Encrypt(Key, hasil XOR) → dapat ciphertext. Ulangi untuk blok berikutnya. ✍ Cara menjelaskan di papan tulis Tulis rumus AES CBC: Enkripsi: C1 = E(Key, P1 ⊕ IV) Ci = E(Key, Pi ⊕ Ci-1) Dekripsi: Pi = D(Key, Ci) ⊕ Ci-1 Gambarkan diagram alur: Panah dari plaintext → XOR dengan IV → AES Encrypt → Ciphertext. Ciphertext[0] → dipakai XOR dengan plaintext[1] → AES Encrypt → Ciphertext[1]. Tunjukkan bahwa IV tetap → mudah dipahami, tapi kurang aman di real-world.
